# 모바일 QA 체크리스트 (PRD-v0.1)

기준일: 2026-02-16
범위: 홈 Hero 캐릭터/요약, 구독 목록, 우측 알림 패널, 파비콘

## 1) 테스트 환경

- 브라우저: Chrome 최신 안정 버전
- 플랫폼: Flutter Web
- 뷰포트(필수):
  - 360x800
  - 390x844
  - 412x915
- 뷰포트(권장):
  - 768x1024

## 2) 기능 체크

- [ ] 홈 첫 진입 시 캐릭터 + 요약이 즉시 보인다.
- [ ] 스크롤 시 Hero 축소 인터랙션이 자연스럽다.
- [ ] 구독 목록에는 캐릭터가 노출되지 않는다.
- [ ] 우측 상단 알림 버튼으로 패널이 열리고 닫힌다.
- [ ] 구독 추가 버튼 동작 후 목록 반영이 정상이다.

## 3) 애니메이션 체크

- [ ] 탭/클릭 반응이 1회 재생되고 중첩 재생되지 않는다.
- [ ] 연속 탭 시 프레임 드랍/깜빡임 없이 동작한다.
- [ ] Idle/State hint가 정보 가독성을 해치지 않는다.

## 4) UI/가독성 체크

- [ ] 텍스트/아이콘 대비가 충분하다.
- [ ] 주요 탭 타깃이 모바일에서 누르기 어렵지 않다.
- [ ] Hero 축소 상태에서도 핵심 요약 문구를 읽을 수 있다.
- [ ] 경고 색(임박 결제)이 과도하지 않고 구분 가능하다.

## 5) 성능 체크

- [ ] 스크롤 + 애니메이션 동시 동작 시 체감 버벅임이 없다.
- [ ] 알림 패널 열고 닫을 때 지연이 체감되지 않는다.
- [ ] 저사양 모바일 에뮬레이션에서도 주요 동선이 유지된다.

## 6) 파비콘 체크 (일반 기준)

- [ ] 16px에서 얼굴 실루엣/눈/코 포인트가 식별된다.
- [ ] 32px/64px에서 윤곽 깨짐 없이 선명하다.
- [ ] 배경/외곽선 대비로 브라우저 탭에서 묻히지 않는다.

## 7) 최종 접근성(A11y) 점검

- [ ] 텍스트 스케일 1.3x/1.5x에서 핵심 문구 잘림이 없다.
- [ ] 상단 액션(알림/구독추가)에 의미 있는 접근성 라벨이 제공된다.
- [ ] 키보드 포커스 이동 시 주요 액션 순서가 논리적으로 유지된다(Web).
- [ ] 주요 인터랙션 타깃이 최소 44x44 기준을 만족한다.
