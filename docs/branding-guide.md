# Subscription Mong Branding Guide v0.1

## Overview
- Purpose: Keep messaging and visual language consistent so users can make faster subscription decisions.
- Scope: PRD-001 / PRD-002 outputs and app screens

## Core Tone
- Trustworthiness: fact-first, not overstatement
- Readability: short sentences, priority on key metrics
- Emotion: empathy allowed, but fear-inducing wording is restricted

## Tone Examples
- Allowed: `Let's check the current status.`
- Allowed: `The renewal date is approaching.`
- Allowed: `Please confirm these items before proceeding.`
- Restricted: `mandatory`, `guaranteed`, `100%`, `you'll lose out if you miss`

## Forbidden / Sensitive Terms
- Overclaiming: `best`, `totally`, `for sure`, `in one shot`
- Fear-based: `it's over`, `ruined`, `damage`
- Misleading: `guaranteed`, `fixed profit`, `never`

## Copy Review Rules
1. For key figures (amount/date/remaining days), always state unit and reference time.
2. One core message and up to two supporting messages per screen.
3. New channels (alerts, guides, notices) must inherit the same label set (tone, intensity, constraints).

## Visual Principles
- Background: light neutral tone with sufficient contrast
- Actions: consistent verbs for the same action across button labels
- Warnings: start with low intensity, emphasize only core states

## Component Rules
- Notification badge: show at most one warning and one confirmation state simultaneously.
- Card: group status text, status metric, and action into one unit.
- Button: prefer verb-first labels, ideally 1-2 words.

## PRD Mapping Rules
- PRD-001: apply branding-priority copy rules to body text and requirements.
- PRD-002: apply first to core screens (Home, Alerts, Subscription Management).
