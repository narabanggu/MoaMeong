# 팀 정의

## 운영 리더십

- 최상위 리더: 사용자(최종 승인권, 착수/중단 최종 결정)
- 2차 리더: 리더멍(운영 총괄, 보고 정리 및 기록 책임)

## 팀 운영 기준(상세는 런북/개별 에이전트 문서 참조)

1. 사용자 노출 문구는 한글로만 작성한다.
2. 내부 식별자/파일명은 영문 표기를 사용한다.
3. MVP 기술 경계: `Flutter Web 단일 앱 + localStorage` 유지, 백엔드/DB/외부 연동 추가는 사용자 승인 전 금지.
4. 승인 흐름은 `체크멍(게이트) -> 가드멍(리스크) -> 리더멍(최종)` 순으로 정합성 문서화한다.
5. 최종 확정은 사용자 승인과 `change log` 반영 후 진행한다.
